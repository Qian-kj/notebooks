1. Introduction

This chapter provides an introduction to the problem of learning languages, focusing specifically on finite automata.It reviews two primary frameworks, efficient exact learning and identification in the limit, for learning automata and presents algorithms for each.

  1. Linguistic and Computational Problems

    Humans naturally learn languages, forming well-structured sentences and judging grammaticality from an early age.In computer science, the problem translates to learning the representation of computational devices generating these languages.

  2. Automata

    Automata provide models for different applications, such as networking, image processing, text and speech processing, logic, etc. They can be simple or efficient approximations for more complex devices.

3. Efficient exact learning

  1. Efficient Exact Learning Framework

    • The efficient exact learning framework focuses on identifying a target concept from a finite set of examples within a polynomial time bound.
    • The objective is to determine the concept exactly, not approximately.
    • This model does not assume that instances are drawn from any unknown distribution.

  2. Concept Class

    • A concept class is efficiently exactly learnable if there exists an algorithm that can learn any concept exactly within polynomial time bounds related to the representation and example sizes.

  3. Learning Scenarios: Passive vs. Active Learning

    • Passive Learning: Similar to the standard supervised learning scenario but without stochastic assumptions. The algorithm passively receives data instances and returns a hypothesis.
    • Active Learning: The learner actively selects training samples through various types of queries, aiming to improve learning efficiency. This scenario involves more interactive participation from the learner.

3.2 Learning with queries

This section discusses the model of learning with queries, involving a minimal teacher or oracle and an active learner. In this model, the learner can make two types of queries to the oracle: membership queries and equivalence queries. The goal is to understand how efficiently a concept class can be exactly learned using these types of queries.

  1. Membership Queries

    • The learner requests the target label of an instance and receives that label.
    • This helps the learner gather specific labels for instances it chooses.

  2. Equivalence Queries

    • The learner proposes a hypothesis and receives feedback on whether matches the target function . If , the learner receives a counter-example.
    • This process helps refine the learner’s hypothesis by pointing out specific errors.

  3. Efficient Exact Learning

    • A concept class is efficiently exactly learnable with membership and equivalence queries if it can be learned within this model efficiently.
    • Such oracles are typically not available in practice, but they provide a useful theoretical framework that can lead to positive results.

4. Identification in the limit

This section discusses the concept of identifying a target concept exactly after receiving a finite set of examples within the identification in the limit framework. A class of languages is identifiable in the limit if there exists an algorithm that can identify any language in that class after examining a finite number of examples, such that its hypothesis remains unchanged thereafter.

  1. Identification in the Limit Framework

    • This framework is less computationally practical as it doesn’t impose an upper bound on the number of instances or the efficiency of the algorithm.
    • It is argued to be similar to human language learning scenarios.
    • In this framework, negative results also apply to the general problem of learning DFAs.

  2. Theorem 16.6

    • Deterministic automata are not identifiable in the limit from positive examples. This implies that there is no algorithm that can perfectly learn DFAs using only positive examples within this framework.

  3. Sub-classes of Finite Automata

    • Some sub-classes of finite automata can be identified in the limit.
    • Most algorithms for inferring automata are based on a state-partitioning paradigm:
      • Start with an initial DFA (usually a tree) accepting the finite set of sample strings and the trivial partition.
      • At each iteration, merge partition blocks while preserving congruence properties until no further merging is possible.
      • The final partition defines the inferred automaton.
